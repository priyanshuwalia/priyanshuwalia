<details>
<summary><strong>&nbsp;⬡ &nbsp;NexaSwap — Decentralised Cross-Chain DEX Aggregator</strong></summary>

<br/>

NexaSwap is a production-grade, cross-chain DEX aggregator that routes swaps across **12 networks** and **40+ liquidity sources**, finding optimal paths through fragmented on-chain liquidity with a single signed transaction. The protocol introduces a novel **intent-based settlement layer** that separates order expression from execution, enabling gasless UX while maintaining full non-custodial guarantees.

<div align="center">

| Attribute       | Detail                                                                                        |
| :-------------- | :-------------------------------------------------------------------------------------------- |
| **Stack**       | Solidity · Rust (solver) · TypeScript · Next.js · viem · The Graph · PostgreSQL · Redis       |
| **Scale**       | $180M+ cumulative volume · 85K+ unique wallets · 12 supported chains                          |
| **Performance** | Sub-200ms quote latency · 99.97% uptime · 0.12% average price improvement vs best single-DEX  |
| **Security**    | Audited by Trail of Bits & Spearbit · 100% invariant test coverage · $250K bug bounty program |
| **Impact**      | Finalist — ETHGlobal 2024 · Integrated by 3 institutional trading desks                       |
| **Repository**  | [github.com/priyanshuwalia-dev/nexaswap](https://github.com/priyanshuwalia-dev/nexaswap)      |

</div>

NexaSwap's solver network operates as a permissioned-then-permissionless auction, where solvers compete on-chain for the right to fill user intents. The smart contract layer is fully upgradeable via a timelocked proxy pattern with on-chain governance. The frontend is a Next.js 14 app with edge-rendered quote pages, leveraging wagmi v2 and viem for type-safe contract interactions and RainbowKit for wallet connectivity. Indexing is handled by a custom Ponder instance replicated across three regions.

<summary><strong>&nbsp;⬡ &nbsp;VaultCore — Institutional On-Chain Asset Management Protocol</strong></summary>

<br/>

VaultCore is a modular, ERC-4626 compliant asset management protocol designed for institutional treasuries and DAO fund managers. It introduces a **strategy composition layer** that allows complex yield strategies to be composed from atomic building blocks — lending deposits, LP positions, staking, and options writing — governed by on-chain risk parameters and configurable rebalancing triggers.

<div align="center">

| Attribute       | Detail                                                                                     |
| :-------------- | :----------------------------------------------------------------------------------------- |
| **Stack**       | Solidity · Foundry · TypeScript · React · Wagmi · FastAPI · Postgres · AWS ECS             |
| **Scale**       | $65M peak TVL · 200+ institutional vault deployments · 14 integrated yield strategies      |
| **Performance** | Gas savings of 34% vs comparable protocols · Automated rebalancing every 6 hours           |
| **Security**    | Audited by Code4rena (no criticals) · 98% branch coverage · Echidna fuzz suite             |
| **Impact**      | Adopted by 3 DAO treasuries managing $40M+ combined · Featured on DeFiLlama                |
| **Repository**  | [github.com/priyanshuwalia-dev/vaultcore](https://github.com/priyanshuwalia-dev/vaultcore) |

</div>

The protocol's architecture separates concerns cleanly across four contract layers: Vault (ERC-4626 accounting), StrategyManager (composition & routing), RiskOracle (parameter enforcement), and FeeCollector (performance & management fees with DAO splits). Each strategy is independently auditable and deployable. The management dashboard is a React SPA with real-time position updates via WebSocket connections to a FastAPI event indexer, backed by a Postgres time-series schema for historical APY tracking.

<details>
<summary><strong>&nbsp;⬡ &nbsp;ChainID — Decentralised Identity & Credential Verification Layer</strong></summary>

<br/>

ChainID is a privacy-preserving, decentralised identity infrastructure built on **EIP-7495 compliant soulbound tokens** and **zero-knowledge credential proofs**. It enables users to prove real-world attributes — KYC status, accredited investor standing, professional credentials — on-chain without exposing underlying personal data, using Groth16 zkSNARK proofs generated client-side.

<div align="center">

| Attribute       | Detail                                                                                          |
| :-------------- | :---------------------------------------------------------------------------------------------- |
| **Stack**       | Solidity · Circom · SnarkJS · TypeScript · Next.js · Node.js · IPFS · AWS Lambda                |
| **Scale**       | 120K+ credential issuances · 18 protocol integrations using ChainID verification gate           |
| **Performance** | Proof generation in <800ms on client · Verification gas cost: ~210K units                       |
| **Security**    | ZK circuit audit by Veridise · Soulbound transfer guards · GDPR-compatible off-chain data layer |
| **Impact**      | Adopted by 2 DeFi protocols for compliant token-gating · Presented at Devcon 7 Bangkok          |
| **Repository**  | [github.com/priyanshuwalia-dev/chainid](https://github.com/priyanshuwalia-dev/chainid)          |

</div>

ChainID's credentialing flow begins with off-chain attestation from authorised issuers (KYC providers, universities, professional bodies), which is hashed and anchored on-chain as a soulbound NFT. Users generate a ZK proof locally in-browser via a WASM-compiled Circom circuit, which is then verified by the on-chain verifier contract for any downstream protocol that integrates the ChainID gate. The system is chain-agnostic and has been deployed across Ethereum mainnet, Polygon, and Base.
